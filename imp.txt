first of all lets get familiar with virtual space address 
in simple words the memory layout or the memory stack whatever you understand or whatever suits you 

now it has 5 sections 
1. text section: the part that contains the binary instructions to be executed by the processor
2. data section: it has some non zero initialized static data
3. bss (block started by symbol): it has zero initialized data.static data uninitialized in program is initialized 0 and goes here 
4. heap: contains dynamically allocated data
5. stack: has variables, func args, cpy of base ptr

stack and heap grows in opposite direction .. means they grow towards each other 

data,bss and heap together are known as data segments
and its end is shown by a system call named brk
it is a pointer nothing else it is called as program break which as the name suggests means program break

when we request system for more memory .. we tell system to increment brk ans similarly when we tell to free memory we tell system to decrement brk

there is one system call named sbrk()
it helps us handle program break
sbrk(0) gives u current addr of program break (where the program breaks)
sbrk(10) -> increment brk by 10 bytes 
sbrk(-10) -> decrement brk by 10 bytes

on failure sbrk returns (void*)-1

you might think thats easy .. we can simply call sbrk with the size bytes and boom we got that memory .. but the tricky part is that free(ptr) func frees the memory block pointed to by the ptr .. which must have been returned by the prev func call to malloc or calloc ..

also heap memory is contiguos .. so we can only release the memory at the end of the heap .. we cant release a block of memory in the middle ... 
so from now .. freeing a block of memory does not mean releasing memory back to os .. it just means  we keep the block marked as free .. this block marked as free may be reused later .. since memory in middle cant be released ..
